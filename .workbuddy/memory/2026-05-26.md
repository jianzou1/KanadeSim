### B1 补丁：渲染插值（22:19）
- 用户反馈"小方块移动卡顿"
- 根因：Worker 4Hz tick，渲染 60Hz，60 帧里 56 帧画同位置，视觉上每秒跳 4 次
- 修复方案：AgentInstances 内部做位置插值
  - prevX/prevZ + currX/currZ 双缓冲
  - 每帧算 alpha = (now - snapReceivedAt) / TICK_MS，clamp [0, 1.2]
  - lerp(prev, curr, alpha)
- API 拆分：applySnapshot → ingestSnapshot(收快照) + renderTick(每帧)
- 主线程 onSnapshot 调 ingestSnapshot，主循环调 renderTick
- 这是 design.md §4 模拟/渲染分离的标准实践，B2 压测前必须有

### 长期约定（追加）
- 任何"低频模拟 + 高频渲染"组合必须做插值，否则视觉跳跃
- 通用模式：模拟数据双缓冲（prev/curr）+ 渲染时基（snapReceivedAt + tickMs）+ alpha clamp

### C3.3 三个修复（23:07）

**1. 瞬移问题（重要 bug fix）**
- 根因：60 tick 重采样会重建 agentStore，新代理位置和旧的完全不同
- 主线程双缓冲（prev/curr）不知道，下一帧 alpha 把旧位置 lerp 到新位置 → 视觉瞬移
- 修复：SimSnapshot 加 respawned 字段
  - Worker 在 spawnVisibleAgents 后下一份 snapshot 标 respawned=true
  - 主线程 ingestSnapshot 检测到 respawned，跳过 prev/curr 拷贝，prev = curr，alpha 任何值都不产生位移
- init 时也算一次 respawn，避免初始的 (0,0) → 真实位置 lerp

**2. 车视觉加大**
- 几何从单个 BoxGeometry(0.5×0.25×0.3) 升级为合并几何
- 车身 0.45×0.32×0.95（更长更高）+ 车顶 0.40×0.18×0.55（驾驶舱）
- 用 three/examples/jsm/utils/BufferGeometryUtils 的 mergeGeometries 合并
- 视觉上明显比步行的 0.22×0.45×0.22 立方体大

**3. 步行只走道路两侧（人行道）**
- planPath 加 sideOffset 参数
- driver 传 0（路中央），walker 传 0.7（路宽 2 的两侧人行道）
