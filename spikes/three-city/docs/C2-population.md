## 2. 经济模型（最小版闭环）

### 2.1 反馈环

```
                    ┌──────────────────────────┐
                    │  总人口 ←→ 总岗位         │
                    │     ↓        ↓            │
                    │  失业率  ←  匹配           │
                    │     ↓                     │
                    │  住宅满意度               │
                    │     ↓                     │
                    │  人口增减（拉回总人口）   │
                    └──────────────────────────┘
                              ‖
                              ▼
              总人口 → 商业需求 → 商业覆盖率 → 商业满意度
                              ‖
                              ▼
              所有建筑 → 按人口产税 → 累计税收
```

### 2.2 核心公式

| 指标 | 公式 |
|---|---|
| 劳动力 | `totalPop × 0.6` |
| 失业率 | `1 - employed / labor` |
| 住房压力 | `totalPop / housingCapacity` |
| 商业需求 | `totalPop × 0.4` |
| 商业覆盖率 | `min(1, commCap / commercialDemand)` |
| 住宅满意度目标 | `1 - 失业率×0.7 - 拥挤惩罚` |
| 住宅人口变化 | `(sat - 0.5) × 2 × (growth or decay rate) × cap` |
| 税收 | `pop × {res:0.02, com:0.05, ind:0.04}` |

### 2.3 数值调参（C2 阶段）

| 参数 | 值 | 含义 |
|---|---|---|
| `HOUSING_GROWTH_PER_TICK` | 0.04 | 满意度满时每 tick 吸引 4% 容量 |
| `HOUSING_DECAY_PER_TICK` | 0.06 | 满意度低时每 tick 流失 6% |
| `SATISFACTION_LERP` | 0.15 | 满意度平滑过渡速度 |
| `COMMERCIAL_DEMAND_PER_CAPITA` | 0.4 | 每市民需要 0.4 商业岗位 |

这些参数刻意做得"反馈快"，方便短时间内看出闭环：
- 一个 60 秒模拟日内能看出人口增长趋势
- 失业/拥挤会在 30 秒内拉低满意度
- 商业不足会立刻显示在 HUD

## 4. 架构变化

### 之前（B1/B2）
```
Worker:
  init(agents) → 随机生成代理（左半住宅、右半工作）
  tick: stepTick(agents)
```

### 现在（C2）
```
Worker:
  init({ buildings, maxVisibleAgents })
   ├ BuildingStore 装载建筑
   ├ seedInitialPopulation 撒种子
   └ spawnVisibleAgents 从住宅/工作建筑采样可见代理
  tick:
   ├ stepEconomy(BuildingStore)   → 更新人口/就业/满意度/税收
   └ stepTick(AgentStore)          → 推进可见代理移动
```

**关键变化**：
- 代理不再是城市的"主体"，建筑才是
- 可见代理只是从总人口里采样的"代表"，专门用来骗肉眼（design.md §5 第二层）

## 6. 演示场景（默认运行）

500 栋建筑随街区分配：
- 4 个住宅街区（容量约 ~8000 居住）
- 4 个商业街区（约 ~3000 岗位）
- 1 个工业街区（约 ~1500 岗位）

初始种子人口：每栋住宅 4 人 = ~600 人。

**预期演化**（约 1-2 分钟）：
1. 种子人口快速繁殖 → 人口涨到 ~3000
2. 接近住房上限时拥挤惩罚启动 → 满意度下降 → 增长放缓
3. 工作岗位被填满 → 商业覆盖率达 100% → 满意度回升
4. 系统进入"刚刚好"的均衡状态，人口在 ~7000-8000 间小幅波动
