## 2. 渲染管线

**核心方案**：场景渲染到低分辨率 RenderTarget → 全屏 quad 用 nearest filter 放大到屏幕。

```
WebGLRenderer (antialias: false, pixelRatio: 1)
    │
    ├── pass 1: scene → RenderTarget (rtW × rtH)        ← 真实渲染负载
    │           NearestFilter / NearestFilter
    │           sRGBColorSpace
    │
    └── pass 2: RT → screen (FSQuad + ShaderMaterial)
                可选 color quantization (按 Q 切换)
```

**为什么这套**：
- 不引入 `EffectComposer` / `postprocessing` 包 → 零额外依赖
- 渲染负载随 RT 分辨率线性下降 → B2 压测可调 pixelScale 验证 GPU 余量
- 像素边缘干净，无 AA 模糊

**互动控制**：
| 操作 | 效果 |
|---|---|
| 左/右键拖拽 | 旋转视角（方位角 + 俯仰角）|
| 滚轮 | 缩放（改 viewSize 4-18）|
| 方向键 | 微调视角 |
| <kbd>1</kbd>/<kbd>2</kbd>/<kbd>3</kbd>/<kbd>4</kbd> | 像素比例 1x/2x/3x/4x |
| <kbd>Q</kbd> | 切换颜色量化（减色）|

## 5. 已知技术债（不阻塞 A2 通过，但记录给 MVP 阶段）

| 项 | 说明 | 优先级 |
|---|---|---|
| pixelScale 切换通过重建管线实现 | 偷懒做法，正式版应在 PixelPipeline 内部支持 setPixelScale() | 低 |
| 建筑没用 InstancedMesh | A2 阶段建筑数 <20，无所谓；B2 必须改 | 中 |
| `wallGeomCache` 共享几何但材质未共享 | 每栋建筑独立 material，B2 压测前要合批 | 中 |
| 没有阴影 | shadowMap 开了会影响性能基线，A2 不开；B2 视情况再加 | 低 |
| 道路用 PlaneGeometry 拼，z-fighting 风险 | A2 用 +0.01/+0.02 偏移，正式版应用 polygonOffset | 低 |
| 雾气和减色同时开效果会冲突 | 减色后雾的渐变会出现条带，A2 不修 | 低 |
